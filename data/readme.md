# Разделение среды и предотвращение утечки данных (Data Leakage).
1. Разделение на X и y (Признаки и Цель)X (признаки) и y (целевая переменная).  
Зачем разделять:   
При обучении модель должна учиться только по X предсказывать y. Если мы случайно прокинем y в X — произойдет утечка данных (Data Leakage), и модель получит нереально высокий, но бесполезный R². Разделение предотвращает это.  
2. Разделение на Train / Validation / Test  
TrainX_train.csv, y_train.csv = Обучение модели (большая часть данных). Модель запоминает здесь закономерности.  
Validation (Val)X_val.csv, y_val.csv = Тюнинг модели (подбор гиперпараметров через RandomizedSearchCV)  
TestX_test.csv, test_ids.csv = Финальная оценка. Ответы FloodProbability для X_test неизвестны.  
test_ids.csv нужен, если бы отправляли на kaggle
